# Frontend Design Comparison

This folder contains before/after screenshots showing the difference between building a component with and without the `/frontend-design` plugin.

## What Goes Here

- `without-plugin.png` — A component built using only the design system tokens (correct but conservative)
- `with-plugin.png` — The same component built with the `/frontend-design` plugin active (bold, distinctive, intentional hierarchy)

These screenshots are referenced in L3 when introducing the design context concept. The comparison demonstrates why design tokens alone produce "technically correct but boring" output, and how the plugin pushes Claude past default conservatism.
